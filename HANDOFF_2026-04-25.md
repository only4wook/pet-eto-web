## 🔥 다음 세션 우선순위 (사용자 신규 요청, 2026-04-25 끝 무렵)

### 1. **갤럭시 사진 업로드 실패 버그** (HIGH 🚨)
**증상**: 갤럭시폰에서 앨범 사진 선택해서 업로드하면:
```
이미지 업로드 실패: Unexpected token 'R', "Request En"... is not valid JSON
카메라로 직접 촬영해서 다시 시도해주세요.
```

**원인 추정 (검증 필요)**:
- 갤럭시 사진은 보통 5~12MB+ (고해상도 4032×3024)
- Vercel serverless body size limit **4.5MB** 초과
- 서버가 `413 Request Entity Too Large` 를 plain text 로 반환
- 클라이언트(`/api/upload-image` 호출하는 곳)가 JSON.parse 시도 → 첫 글자 'R'(equest) 만나 실패
- 그래서 catch 가 "카메라로 직접 촬영" 폴백 메시지 노출

**해결안**:
1. **클라이언트 자동 압축** (`src/app/feed/upload/page.tsx`):
   - File 받자마자 canvas 로 리사이즈 → max 1920px + JPEG 80%
   - 기존에 admin 페이지에 이미지 압축 코드 있음 (`src/app/admin/page.tsx`) — 재사용
2. **HEIC 처리**: 이미 서버에 HEIC 변환 로직 있음 (commit fecd0c1) — 갤럭시는 보통 JPG라 무관할 수도
3. **Storage 직접 업로드 우회**: 큰 파일은 Supabase Storage signed URL 로 직접 업로드 → /api/upload-image 우회
4. **에러 메시지 개선**: JSON.parse 실패 시 "사진이 너무 커요. 1장당 4MB 이하로 줄여주세요." 처럼 명확히

**파일**:
- `src/app/feed/upload/page.tsx` (클라이언트)
- `src/app/api/upload-image/route.ts` (서버)
- `src/app/admin/page.tsx` (압축 코드 참고용)

### 2. **여러 장 사진 업로드** (MEDIUM ✨)
**현재**: 사진 1장만 업로드 가능 (`<input type="file">` 단일)
**요청**: 인스타처럼 N장 업로드

**구현**:
1. `<input type="file" multiple accept="image/*">` 변경
2. 미리보기: thumbnail 그리드 (1×1, 2×1, 2×2, 3×3)
3. DB 스키마: `feed_posts.image_url` (TEXT) → `image_urls JSONB` (배열) — 또는 새 테이블 `feed_images`
4. AI 분석: 첫 1장 분석 + "추가 사진 N장" 표시 OR 모든 사진 분석 후 종합 진단 (추천)
5. 표시: 피드 카드/상세에서 swipeable carousel
6. 동영상은 그대로 1개

**스키마 변경 SQL** (작성 필요):
```sql
ALTER TABLE feed_posts ADD COLUMN image_urls JSONB DEFAULT NULL;
-- image_url 컬럼은 호환성 위해 유지 (첫 장만 저장하거나 NULL)
```
